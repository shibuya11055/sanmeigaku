# /fortune_analysis 運勢解説参照パネル計画

## 背景

`/fortune_analysis` は、生年月日と性別から命式・陽占・三期エネルギー・宿命位相・年運・大運などを表示する鑑定画面である。現状でも、位相名や宿命天中殺などをクリックすると右側のサイドパネルで解説を参照できる。

`materials/中国五行占術_算命学⑦回.md` には、年運・月運・日運・大運の読み方、運勢陽占、運勢エネルギー、運勢位相法、天中殺、条件別占技がまとまっている。この知識を、鑑定結果の文脈に沿って素早く参照できるようにする。

注意: `materials` 配下の本文は権利物の可能性があるため、画面表示では原文転載を避け、要約・再編集した辞書データとして扱う。

## ゴール

- 占い師が年運表・大運表の結果を見ながら、該当する解説をすぐ参照できる。
- 単なる用語説明ではなく、「この年・この大運をどう読むか」の形で鑑定補助になる。
- 既存の右サイドパネル UI と Stimulus 構成を活かし、画面遷移なしで参照できる。

## 対象画面

- `/fortune_analysis`
- `app/views/fortune_analysis/_yearly_fortune.erb`
- `app/views/fortune_analysis/_major_fortune.erb`
- `app/views/fortune_analysis/_detail.erb`

## 既存機能との接続点

既存では以下のようなサイドパネル系機能がある。

- 位相用語: `relationship-panel`
- 宿命位相法: `natal-phase-panel`
- 三期エネルギー: `energy-panel`
- 宿命天中殺 / 運勢天中殺: `birth-void-panel`
- 異常干支: `abnormal-panel`
- 宿命五型: `beast-panel`

今回の追加は、これらと同じ UI パターンを使い、年運・大運の「行」や「セル」から運勢解説を開く方向が望ましい。

## 機能候補

### 1. 年運表の1行クリックで「年運読み解き」パネル

年運表の1行を選択すると、その年の総合的な読み解きを右サイドパネルに表示する。

表示候補:

- 年・年齢
- 巡る干支
- 主星: その年のテーマ
- 従星: その年を支えるエネルギー
- 日支位相: 家庭・結果・私生活
- 月支位相: 心の状態
- 年支位相: 仕事・社会・学校
- 天中殺の有無
- 鑑定メモ用の短いまとめ

プロダクト価値:

- 年運表の情報を個別に読む負担を減らせる。
- 「主星、従星、日/月/年の位相を統合して読む」というドキュメントの思想を画面上で再現できる。
- 鑑定中の説明がしやすくなる。

MVP では、現在表示済みの `yearly_fortune` データから組み立てる。新しい計算ロジックは最小限にする。

### 2. 位相セルの場所別解説を強化

既存の位相セルは `半会`、`支合`、`害` などの用語説明を出せる。これを、年運・大運の列位置に応じて説明を出し分ける。

場所別の意味:

- 日支: 家庭、結果、私生活
- 月支: 心、自我、精神状態
- 年支: 仕事、社会、学校、行動

例:

- `日支 半会`: 家庭や結果がまとまり、願い事が形になりやすい。
- `月支 半会`: 心が拡大し、成長感や充実感が出やすい。
- `年支 半会`: 仕事や人間関係が広がり、発展しやすい。
- `日支 害`: 家庭や結果に負担が出やすく、家族への奉仕が鍵になる。
- `月支 害`: ストレスや内的葛藤が強まりやすい。
- `年支 害`: 仕事や社会で受け身になりやすく、奉仕的対応が鍵になる。

プロダクト価値:

- 現状の用語辞書より鑑定実務に近くなる。
- 既存 UI の自然な拡張で実装しやすい。
- 年運表・大運表の `日/月/年` 列の意味が明確になる。

データ候補:

- `config/relationship_glossary.yml` を拡張する。
- あるいは `config/fortune_relationship_glossary.yml` のように運勢専用辞書を新設する。

### 3. 年運・大運の主星クリック解説

年運表・大運表の `主星` をクリックできるようにし、運勢として巡った十大主星の意味を表示する。

対象:

- 貫索星
- 石門星
- 鳳閣星
- 調舒星
- 禄存星
- 司禄星
- 車騎星
- 牽牛星
- 龍高星
- 玉堂星

表示候補:

- キーワード
- キー人物
- 通常時の運勢意味
- 天中殺時の注意
- 鑑定時の短い問いかけ

例:

- `鳳閣星`: 遊び、自然体、目下・子供、表現、リフレッシュ。
- `貫索星`: 自分、独立心、守り、頑固さ、スタート。
- `禄存星`: 愛情、財、奉仕、異性縁、出費。

注意:

既存の陽占・宿命解説と混同しない。ここでは「運勢に巡った時の意味」として扱う。

### 4. 年運・大運の従星クリック解説

年運表・大運表の `従星` をクリックできるようにし、運勢エネルギーとして巡った十二大従星の意味を表示する。

対象:

- 天報星
- 天印星
- 天貴星
- 天恍星
- 天南星
- 天禄星
- 天将星
- 天堂星
- 天胡星
- 天庫星
- 天極星
- 天馳星

表示候補:

- パワー点数
- キーワード
- 通常時の運勢意味
- 天中殺時の注意

注意:

既存の `twelve_sub_star_button` は三期エネルギー向けで、若年期・中年期・晩年期の文脈を持つ。年運・大運では別文脈なので、運勢専用の辞書・ボタンに分けるのが望ましい。

### 5. 今年の主役人物表示

運勢主星から、その年にテーマになりやすい人物を表示する。

例:

- 貫索星 / 石門星: 自分、兄弟、仲間、仕事関係者
- 鳳閣星 / 調舒星: 子供、目下、後輩
- 禄存星 / 司禄星: 恋人、配偶者、家族、補佐役
- 車騎星 / 牽牛星: 異性、配偶者、他人、補佐役
- 龍高星 / 玉堂星: 親、目上、上司、先生

プロダクト価値:

- 鑑定者が「今年誰との関係を見るか」をすぐ把握できる。
- 年運表の主星が、実際の相談テーマに接続しやすくなる。

表示場所:

- 年運読み解きパネル内
- あるいは年運表の行詳細として表示

### 6. 条件別タイミング診断

ドキュメントの「結婚・就職・独立・転居に適した運勢」を、年運表から候補年として抽出する。

対象イベント:

- 結婚
- 就職 / 転職
- 独立 / 開業
- 転居

基本方針:

- 天中殺年は原則避ける。
- 結婚は日支の位相を重視する。
- 就職・転職・独立は年支の位相を重視する。
- 転居は住居なら日支、仕事場なら年支を重視する。

表示例:

- 「結婚に向く年」候補を一覧表示。
- 候補年をクリックすると、なぜ適しているかをサイドパネルで説明。
- 不向きな年についても、理由を簡潔に表示。

プロダクト価値:

- 鑑定中によく聞かれる「いつが良いか」に直接答えられる。
- 年運表を見るだけでは見落としやすい候補年を拾える。
- SaaS として差別化しやすい。

実装は MVP には含めず、第2フェーズ以降が望ましい。条件判定の精度が鑑定結果に強く影響するため、テストとドメインレビューが必要。

### 7. 大運表の10年テーマパネル

大運表の1行をクリックすると、その10年のテーマを表示する。

表示候補:

- 旬
- 干支
- 年齢
- 年代
- 主星
- 従星
- 日支位相
- 月支位相
- 年支位相
- 大運天中殺
- 異常干支 / 変剋律
- 10年単位の読み方

プロダクト価値:

- 大運は人生全体の骨格を読むため、年運よりも鑑定価値が高い。
- 年運だけではなく「今の10年の背景」を説明しやすくなる。

注意:

現在の大運表では `変剋律` 列に単独の異常干支を `異` として表示している。ドキュメント上の変剋律は「大運に異常干支が2つ以上続くとき」が条件なので、以下のように分けるべき。

- `異常干支`: その大運干支が異常干支である。
- `変剋律`: 連続条件を満たして変剋律が成立している。

## MVP 推奨範囲

最初に作るべき範囲は次の3つ。

1. 年運表の1行クリックで「年運読み解き」パネル
2. 年運・大運の位相セルを場所別解説に強化
3. 年運・大運の主星 / 従星クリック解説

理由:

- 既存の年運表・大運表のデータをそのまま利用しやすい。
- 既存の右サイドパネル UI パターンに乗せられる。
- 鑑定中の利用頻度が高い。
- 条件別診断よりも判定ロジックのリスクが低い。

## 実装ステップ案

### Phase 1: 辞書データ準備

- `materials/中国五行占術_算命学⑦回.md` から、運勢主星・運勢従星・運勢位相を要約する。
- 原文をそのまま入れず、画面表示向けに短文化する。
- YAML 辞書を追加する。
- 対応する glossary クラスを `lib/` に追加する。

### Phase 2: セル単位のクリック解説

- 年運表・大運表の `主星` をボタン化する。
- 年運表・大運表の `従星` をボタン化する。
- 位相セルに `day/month/year` の位置情報を渡し、場所別解説を表示する。
- 既存パネルと衝突しないよう、`fortune-panel:open` イベントを使う。

### Phase 3: 年運1行パネル

- 年運表の行に詳細ボタン、または行クリックを追加する。
- 総合パネルを新設する。
- 「テーマ」「エネルギー」「家庭/心/仕事」「天中殺」「主役人物」をまとめて表示する。

### Phase 4: 大運1行パネル

- 大運表の行に詳細ボタン、または行クリックを追加する。
- 10年単位の読み方として、年運とは別の見出しにする。
- `異常干支` と `変剋律` の表示ロジックを見直す。

### Phase 5: 条件別タイミング診断

- 結婚、就職、独立、転居の候補年抽出ロジックを追加する。
- 判定理由をサイドパネルで説明する。
- ドメインレビューとテストを厚めに入れる。

## UI 方針

- 既存の右サイドパネルを踏襲する。
- 年運表・大運表は情報密度が高いため、セル内のボタンは小さくしすぎない。
- 行クリックだけにすると誤操作が起きやすいため、MVP では `詳細` アイコンまたはテキストボタンを置くのが安全。
- 既存の色分けは維持し、解説パネル側で「日支 / 月支 / 年支」の文脈を明示する。
- 天中殺時は通常解説と中殺解説を混在させず、明確に「天中殺時の読み」として表示する。

## テスト方針

- glossary lookup の単体テスト
- helper が辞書あり・辞書なしで正しく HTML を返すテスト
- 年運表・大運表にボタンが表示される view/helper テスト
- 条件別タイミング診断を実装する場合は、代表パターンの unit test
- 変剋律を実装または修正する場合は、異常干支単独と連続成立を分けたテスト

## リスクと注意点

- 原文転載リスク: `materials` の文章は要約・再編集して使う。
- 宿命用の意味と運勢用の意味が混同されるリスクがある。
- 天中殺時の解釈は強い表現になりやすいため、UI 文言は実務で使える穏当な表現に整える。
- 条件別タイミング診断は便利だが、判定ミスの影響が大きい。MVP では避け、辞書参照を先に作る。
- 大運の `変剋律` 表示は現状の `異` と意味がずれている可能性があるため、実装前に整理する。

## 優先順位

1. 位相セルの場所別解説
2. 年運・大運の主星 / 従星クリック解説
3. 年運1行の総合読み解きパネル
4. 大運1行の10年テーマパネル
5. 条件別タイミング診断
