# 公式サイトとの命式比較テスト

`Sanmeigaku::SiteComparison` は、算命学総本校・髙尾学館の「Yourself 星出し」と、アプリのDB非依存計算を比較するための補助機能です。

## 使い方

まず、比較対象日を決定的に生成します。既定では、月入日前後の境界日を優先し、乱数で不足分を補って1,000件作成します。

```bash
bin/rails 'sanmeigaku:site_comparison:generate[1000,20260718]'
```

次のタスクは外部サイトへアクセスします。利用許可、アクセス頻度、利用規約を確認したうえで、必要な場合だけ実行してください。既定の待ち時間は1秒です。

```bash
bin/rails 'sanmeigaku:site_comparison:fetch[1000,20260718,1.0]'
```

結果は `tmp/sanmeigaku_site_comparison/results.json` に保存されます。JSONには日付、ローカル計算結果、サイトのJSONレスポンス、比較結果が含まれるため、2回目以降は外部サイトへアクセスせずに再検証できます。

```bash
bin/rails 'sanmeigaku:site_comparison:report'
bin/rails 'sanmeigaku:site_comparison:verify'
```

`verify` は保存済みJSONの日付を使ってローカル計算を再実行し、保存されたサイト結果と比較します。日付の追加や計算ロジックの変更があった場合の回帰確認に利用できます。

## 比較対象

- `insen`: 日干・月干・年干・日支・月支・年支・日蔵干・月蔵干・年蔵干
- `yousen`: 公式サイトの画面表示順（北・右上・左・中央・右・左下・下・右下）
- `tentyusatsu`: 日柱の天中殺

公式サイトの入力画面には性別がないため、大運の比較はこのタスクの対象外です。また、このリポジトリの計算範囲に合わせ、比較日付は1925年2月1日〜2044年12月31日に制限しています。

公式サイトとの不一致は、直ちにロジックバグとは判定しません。高尾式と朱学院式などの流派差、節入日・年表データの差、公式サイト側の境界処理差を切り分けてから修正します。静的マスタ値は別リポジトリのJS側と照合しますが、計算ロジックはこのリポジトリの高尾式を使用します。

### 1964年9月7日の不一致メモ

1,000件比較で確認された1964年9月7日の差は、高尾式と朱学院式の計算ロジック差ではなく、白露の日時を日付マスタへ落とす方法の差と考える。Railsと `yang_center_star_js` は1964年9月の月入日を8日として共有しているため、9月7日は壬申、9月8日から癸酉となる。一方、[国立天文台の1964年暦要項](https://eco.mtk.nao.ac.jp/koyomi/yoko/pdf/yoko1964.pdf) は白露を「9月7日24時40分」と掲載しており、通常時刻では9月8日0時40分相当である。日付だけを扱う資料では9月7日と記録され得る。

[算命学Stockの説明](https://sanmei-stock.com/prior-knowledge/formulas-do-not-match/) でも、NASA相当の天文計算と古い命式手帳・学校資料では節入り日がずれる場合があると説明されている。したがって、現状のRailsマスタを単独で8日から7日へ変更しない。

## テスト

JSON処理と比較ロジック自体は、外部アクセスを行わないMinitestで検証します。

```bash
bin/rails test test/services/sanmeigaku_site_comparison_test.rb
```
